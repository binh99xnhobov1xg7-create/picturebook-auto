## 0. 一句话定位

**Dino Online Club（0-6 级）是一整套以 Mia 和 Tommy 这对兄妹为核心主角的连续系列读物。**
不管单本标题/主题是什么，**每一本书都是这对兄妹的故事**，一切围绕他俩展开。

---

## 1. 主角铁律（最高优先级，不可违背）

1. **Mia（妹妹）+ Tommy（哥哥）是贯穿全系列每一本书的主人公。** 每本书都必须出现他俩。
2. **形象随级别年龄锁死**（"形象" = 脸型 + 五官 + 发型/发色 + 服装款式与配色 + **身高/身材比例**）：

   | 级别 | 主角年龄 | 定妆 key |
   |---|---|---|
   | L0 / L1 / L2 | **8 岁** | `mia_8` / `tommy_8` |
   | L3 / L4 | **10 岁** | `mia_10` / `tommy_10` |
   | L5 / L6 | **12 岁** | `mia_12` / `tommy_12` |

3. **Mia 识别核心**：亚洲女孩，棕色长发**单束高马尾 + 齐刘海**（永远扎马尾，不散开），专属色**紫色**，
   不戴任何饰品/眼镜。服装随年龄（见代码 manifest，勿擅改）。
4. **Tommy 识别核心**：亚洲男孩，棕色**蓬松短发**（绝不长发/马尾），专属色**蓝色**，不戴眼镜/帽子。
5. **同框不分身**：同一个角色在一张画面里只能出现一次，严禁两个 Mia / 两个 Tommy / 分身 / 镜像 /
   既在前景又在背景。
6. **身高即年龄**：8 岁是低龄学童、10 岁是学龄儿童、12 岁是青春期前少年（身形修长个子高），
   身高与身材比例必须符合该年龄，绝不能把 10 岁画成 12 岁的样子。

---

## 2. 配角规则

- 配角**只在被点名或剧情明确需要时**出现，且必须 IP 一致、**绝不抢主角位**。
- 当某本书出现一个特定角色（如 **Anna**），**另外两个主角仍然是 Mia 和 Tommy**（对应年龄）。
- 故事里出现"妈妈/爸爸/爷爷/奶奶/家人" → 这是一家人的写实故事，**配套的孩子必然是 Mia & Tommy**。
- **不硬塞第二位兄妹（用户拍板 2026-06-08）**：当一本书的主角是**某一位兄妹 + 一个具名的人类朋友**
  （如《Mia and Her Spanish Friend》= Mia + Lucia），则**只带被点名的那位兄妹**（Mia），
  **绝不再硬塞 Tommy**——除非故事确实把第二位兄妹也点了名。判定见
  `cn_prompt_builder._lead_keys_for_book` / `_recurring_human_friend_keys`（动物朋友不触发，走框架寓言）。
- **不许凭空新增有清晰五官的陌生小孩/同学/路人**；远景至多一两个极淡人影剪影。

已登记配角（精确形象见代码 SSOT）：
`anna`（绿毛衣+黑 bob+白发箍，配角）、`cate`、`ali`、`mom`、`dad`、`grandma`、`grandpa`、
`teacher`(Teacher Kim)、`winnie`(灰虎斑猫)、`max`(柯基犬)、多元化 `black_girl`/`black_boy`。

## 3. 品牌角色 Dino（特别规则）

- **Dino 不作为故事角色进入任何画面**（模型一画就崩）。仅在 UI / 角落作品牌 logo。
- 这是**所有级别**的硬规则。

## 4. 角色身份「四层判定」（所有书通用·底层铁律）

> 任何一页"谁在画面里、画成谁"，按优先级从上往下判定。这是全系列通用方法，新书直接套用。

1. **① 系列 IP 定妆锁**：Mia / Tommy / 家人 / Teacher Kim / Winnie / Max 等——有官方定妆图，
   按级别年龄档（8/10/12）死锁脸/发/衣/配色/身高。最高优先。
2. **② 官方每课 prompt「在场名单」为权威**：官方已写明本页谁在场（含 "background <10%"
   "not in the frame"）。选角以官方点名为准；**剔除否定句**（"X is not in the frame"），
   **只认专名**（Ms./Teacher Kim，不认裸词 teacher）。
3. **③ 一次性 / 非 IP 角色 = 永远是它自己**（nurse / Goldilocks / 狐狸 / Ravi / Farmer Tom…）：
   **绝不映射成 Mia/Tommy**。反复出场（≥2 页）→ 见 §4.1「书内角色册」做书内锁。
4. **④ 真正无名的泛指小孩**（"a boy"/"a girl"，没名字）→ girl→Mia / boy→Tommy（保证每个孩子都是 IP）。
5. **⑤ 动物 / 反派** → 见 §5「儿童向柔化」，画成 friendly 可爱，绝不凶相。

### 4.1 书内角色册（一次性角色的"书内锁"）

- 出每本书前，自动登记**反复出场的非 IP 角色**，为其：① 锁一句**全书统一外观描述**（每次出现都复用）；
  ② 生成一张**书内定妆锚图**，后续每页用它当 img2img 锚 → 与主角同级锁死，但只在本书内有效。
- 只出现 1 页的配角（路人、护士等）天然一致，只注入统一描述、不生成锚图。
- 实现：`scripts/book_cast.py`（有官方 prompt 用其加粗角色识别；新书只有正文则回退用正文专名/角色名词识别）。

### 4.2 框架式寓言（Mia/Tommy 是"读者"不是故事角色）—— 默认 **方案 B**

- **判定**：有反复出场的非 IP 寓言主角 + 文本含"读书/幻象"框架标记；新书纯动物寓言（无人类主角/家人）
  也自动按框架处理。`scripts/cn_prompt_builder.py` → `_is_frame_fable`。
- **铁律**：兄妹是**读者**，**绝不进入故事、绝不与寓言角色同场互动**。
- **默认呈现 = 方案 B（用户拍板 2026-06-07 锁定）**：每页"读者一角看书 + 故事发光幻象"，
  **幻象是画面主体占约 75%（60–80%）并铺满画面**，读者缩在**左下角约 20–25%**，清晰但不抢戏，
  幻象与读书区用一道细光晕过渡，**除读者那一角外不留大片空白纸面**。
- 备选模式（代码支持，非默认）：A=封面读书引子+内页纯故事；A+=封面+首页开场+末页收尾+中间纯故事。
- 寓言自身的非人角色（饼干人、三只熊、狐狸等）按本色画 + 儿童向柔化，与主角清晰区分。

### 4.3 非虚构 / 科普书（Mia/Tommy = 封面系列 IP，正文内页不出场）—— 用户拍板 2026-06-11

- **判定**：`fiction_type="non-fiction"` 或文本为科普/历史说明体；逐页判定见
  `scripts/cn_prompt_builder.py` → `_nf_body_page` / `_is_nonfiction`。
- **铁律**：Mia/Tommy **只作为连续系列 IP 出现在【封面】**；**所有正文内页一律不出场**
  （不挂主角定妆锚图、不写"主角在做某事"），画面主体 = 本页科普对象/知识过程，按真实比例呈现。
  封面照常带系列 IP（按级别年龄的棕发定妆锁）。
- **年代锁**：每个正文页注入【时代/年代锁】并覆盖冲突的"现代"环境提示——过去页旧式样貌、
  对比页今昔并置、现代页当代样貌；同年代各页布景连贯（实现：`_era_*` / `page_eras`）。
- **自检豁免**：非虚构正文页自动豁免"必须含主角"门（与框架寓言同机制）。
- **判定优先级**：纯动物寓言→框架(§4.2)；非虚构科普→本节(§4.3)；其余真实兄妹/家庭故事→双主角(§1)。

## 4.5 故事连续性层（画面必须跟住故事的连续设定）

- 出图前自动扫全书**天气/时间线索**（雨、雪、夜），找出"同一场天气"的页区间，
  **把它贯穿到区间内所有户外页**——杜绝"同一场雨里中间页突然放晴"。主角真正进洞/室内遮蔽的页自动排除。
- **画面内容始终以"官方逐页场景"为权威源**（已写明每页发生什么、谁在场），再叠加这层连续性保证不跳戏。
- 实现：`scripts/cn_prompt_builder.py` → `_weather_arc` / `_continuity_note`。

---

## 5. 画面质量与画风

- **干净、平滑、治愈系水彩**：柔和晕染、克制精致的明暗体积、清晰细墨线、暖米低饱和。
- **明确拒绝**：脏色斑 / 破碎色块 / 噪点颗粒 / 灰扑扑脏块 / 模糊边缘。即便官方文字提到"纸纹/笔触"，
  也以"干净平滑、可高分辨率印刷"为准（对齐 GPT-4 稳定输出的洁净观感）。
- 有空间纵深（前中后景 / 景深），反扁平、反呆板正面平视；但仍是水彩，不滑向油画/塑料 3D。
- 出图用**图生图**：每个出场 IP 用官方定妆图驱动锁脸，只改姿势/表情，其余不动。

## 5.5 出图前自检门（强制·所有书）

> 每页在出图前自动校验主角铁律，违规累计并**标记"人工抽查"**（非致命，不阻断批量）：

1. 同一角色不得重复（防分身）；2. Dino 不进画面；3. 应含主角的页面必须含本书判定出的系列主角（双主角；或 Mia+具名朋友的书只需 Mia）；
4. Mia/Tommy 参考图年龄档 = 级别对应；5. IP/注册角色 ≤ 3 人（一次性角色不计入）。
框架寓言纯故事页（读者刻意不入场）自动豁免"必须含主角"。实现：`cn_prompt_builder.validate_page_ip_lock`。
